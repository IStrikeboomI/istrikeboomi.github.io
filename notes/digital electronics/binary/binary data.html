When we store binary in a computer, we have to store it as binary data

This because if we have a piece of binary like $0b11100010$, there are many ways to interpret this

If this is Two's Complement number, it would have a different value compared to a regular Binary number

To differentiate these we have to use data types and data sizes

Data Sizes

Each binary digit is called a bit, data is stored in 8 bit chunks called bytes

1 byte = 8 bits

Subsequent sizes use SI/metric prefixes

  1. 1 kilobyte (kb) = 1000 bytes
  2. 1 megabyte (mb) = 1000 kilobytes
  3. 1 gigabyte (gb) = 1000 megabytes
  4. 1 terabyte (tb) = 1000 gigabytes

or

  1. 1 kibibyte (kib) = 1024 bytes
  2. 1 mebibyte (mib) = 1024 kibibyte
  3. 1 gibibyte (gib) = 1024 mebibyte
  4. 1 tebibyte (tib) = 1024 gibibyte

However, there is sometimes confusion because some people use kilo/mega/giga/tera as 1024 instead of 1000

For example, memory drive manufacters say a 1 tb drive is 1,000,000,000,000 bytes

But windows OS says 1 tb is 1,073,741,824 bytes (1024 * 1024 * 1024) bytes, so a 1 tb drive shows up as 931 gb

Data Types

Whenever we store binary data using code, we need some context on how to intrepret said data

We can achieve this using data types

Here are some of the most common ones, commonly seen in programming languages like c++

We haven't covered how all these work through notes yet but we will

For storing integers (whole numbers):

Unsigned (always positive) intgers:

  1. unsigned byte (1 byte/8 bits) (ranges from 0 to 255)
  2. unsigned short (2 bytes/16 bits) (ranges from 0 to 65535)
  3. unsigned int (4 bytes/32 bits) (ranges from 0 to 4,294,967,295)
  4. unsigned long (8 bytes/64 bits) (ranges from 0 to 18,446,744,073,709,551,615)

Signed (can be negative or positive using Two's Complement) intgers:

  1. byte (1 byte/8 bits) (ranges from -128 to 127)
  2. short (2 bytes/16 bits) (ranges from −32768 to 32767)
  3. int (4 bytes/32 bits) (ranges from −2,147,483,648 to 2,147,483,647)
  4. long (8 bytes/64 bits) (ranges from −9,223,372,036,854,775,808 to 9,223,372,036,854,775,807)

For storing numbers with decimals (fractional) using IEEE-754:

  1. float (4 bytes/32 bits) (ranges from $\pm 1.175494351 \cdot 10^{-38}$ to $\pm 3.4028234 \cdot 10^{38}$)
  2. double (8 bytes/64 bits) (ranges from $\pm 2.2250738585072014 \cdot 10^{-308}$ to $\pm 1.7976931348623157 \cdot 10^{308}$)

For storing text characters:

  1. char (1 byte/8 bits) (ASCII character)
  2. char (1 byte/8 bits) (ASCII character)

Binary Coded Decimal (BCD) is a way to represent decimal numbers in binary form

In BCD, each decimal digit is represented by a 4-bit binary number

For example, the decimal number 45 would be represented in BCD as:

4 = 0100 and 5 = 0101, so 45 in BCD is $0100 \ 0101$

The benefit of BCD is that it makes for easier conversion with decimal and binary

Conversion

To convert a decimal number to BCD, we convert each digit to its 4-bit binary equivalent

To convert a BCD number to decimal, we reverse the process

The conversion of a digit to binary is the same as in binary, you can use the table

$$\begin{align} \begin{array} {c | c} \text{Decimal} & \text{BCD (4-bit Binary)} \\ \hline 0 & 0000 \\ 1 & 0001 \\ 2 & 0010 \\ 3 & 0011 \\ 4 & 0100 \\ 5 & 0101 \\ 6 & 0110 \\ 7 & 0111 \\ 8 & 1000 \\ 9 & 1001 \\ \end{array} \end{align}$$

Addition in BCD

You can add BCD numbers normally using binary addition, but there is a big caveat

Since a digit can't exceed 9, if the sum of two BCD digits is greater than 9

You must add a 6 (0110) to each digit, to force an overflow/carry

This is because in standard decimal addition, adding two numbers larger than 9 carries

For example, adding 7 (0111) and 8 (1000) in BCD:

$$\begin{align} 0111 \ (7) \\ + 1000 \ (8) \\ \hline 1111 \ (15) \\ \end{align}$$


Another example 423 + 219 (= 642) in BCD:

$$\begin{align} 1 \ 1 \ \ \ \ \ \ \ \ \ \ \ \ \\\ 0100 \ 0010 \ 0011 \ (423) \\ + 0010 \ 0001 \ 1001 \ (219) \\ \hline 0110 \ 0011 \ 1100 \ \ \ \ \ \ \ \ \ \ \\ \end{align}$$

But there's an issue, the first digit is over 9 so we add 6 to the first digit

$$\begin{align} 1 \ 1 \ 1 \ 1 \ \ \ \ \ \\ 0110 \ 0011 \ 1100 \\ + 0000 \ 0000 \ 0110 \\ \hline 0110 \ 0100 \ 0010\\ \end{align}$$

So the final result is 642 in BCD


Another example 678 + 535 = 1213

$$\begin{align} 0110 \ 0111 \ 1000 \ (678) \\ + 0101 \ 0011 \ 0101 \ (535) \\ \hline 1011 \ 1010 \ 1101 \ \ \ \ \ \ \ \ \ \ \end{align}$$

All 3 digits are over 9, so we add 6 to each digit

$$\begin{align} 1011 \ 1010 \ 1101 \\ + 0110 \ 0110 \ 0110 \\ \hline 0001 \ 0010 \ 0001 \ 0011 \\ \end{align}$$

The final result is 1213 in BCD

Practice Problems
  • Convert $13527$ to BCD

    $$\begin{align} 0001 \ 0011 \ 0101 \ 0010 \ 0111 \\ \end{align}$$

  • Convert BCD $0011 \ 0111 \ 0110 \ 0000 \ 0001$ to decimal

    $$\begin{align} 37601 \end{align}$$

  • Add $1442$ and $72093$ in BCD

    $$\begin{align} 0001 \ 0100 \ 0100 \ 0010 \ (1442) \ \ \\ + 0111 \ 0010 \ 0000 \ 1001 \ 0011 \ (72093) \\ \hline 0111 \ 0011 \ 0100 \ 1101 \ 0101 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \end{align}$$

    The second digit is invalid so we add 6 to it

    $$\begin{align} 0111 \ 0011 \ 0100 \ 1101 \ 0101 \\ + 0000 \ 0000 \ 0000 \ 0110 \ 0000 \\ \hline 0111 \ 0011 \ 0101 \ 0011 \ 0101 \\ \end{align}$$

    So the answer is $1442 + 72093 = 73535$ in BCD.

Hexadecimal (or hex for short) is a base-16 number system that uses 0-9 and then a-f

a represents 11, b represents 12, c represents 13, d represents 14, and f represents 15

Either lower or uppercase characters can be used, just make sure to stay consistent

Hex numbers typically start with 0x

Hexadecimal is beneficial for compacting binary easily

4 binary bits equals 1 hexadecimal digit

Conversion Table

This table is good to memorize, but it can be derived easily

$$\begin{align} 0000_2 \ (0_{10}) = 0_{16} \ \ &\ \ \ \ 1000_2 \ (8_{10}) = 8_{16} \\ 0001_2 \ (1_{10}) = 1_{16} \ \ &\ \ \ \ 1001_2 \ (9_{10}) = 9_{16} \\ 0010_2 \ (2_{10}) = 2_{16} \ \ &\ \ \ \ 1010_2 \ (10_{10}) = A_{16} \\ 0011_2 \ (3_{10}) = 3_{16} \ \ &\ \ \ \ 1011_2 \ (11_{10}) = B_{16} \\ 0100_2 \ (4_{10}) = 4_{16} \ \ &\ \ \ \ 1100_2 \ (12_{10}) = C_{16} \\ 0101_2 \ (5_{10}) = 5_{16} \ \ &\ \ \ \ 1101_2 \ (13_{10}) = D_{16} \\ 0110_2 \ (6_{10}) = 6_{16} \ \ &\ \ \ \ 1110_2 \ (14_{10}) = E_{16} \\ 0111_2 \ (7_{10}) = 7_{16} \ \ &\ \ \ \ 1111_2 \ (15_{10}) = F_{16} \\ \end{align}$$

Converting Between Binary

The way we would traditionally convert to binary would be to divide by 16

But we know by Binary Tricks, diving by 16 is the same as shifting by 4 bits

We can split the binary into chunks of 4 bits and convert each bit to hex

We can also do this process in reverse to convert from hex to binary

Example: convert $0b10101011110101001011000011$

$$\begin{align} 0b10101011110101001011000011 \rightarrow 0010 \ 1010 \ 1111 \ 0101 \ 0010 \ 1100 \ 0011 \end{align}$$

Notice since we only had 26 bits, we had to pad it to 28 bits so we could evenly split by 4

$$\begin{align} 0010 &\rightarrow 2\\ 1010 &\rightarrow A\\ 1111 &\rightarrow F\\ 0101 &\rightarrow 5\\ 0010 &\rightarrow 2\\ 1100 &\rightarrow C\\ 0011 &\rightarrow 3 \end{align}$$

$$\begin{align} 0b10101011110101001011000011 = 0\text{x}\text{2AF52C3} \end{align}$$

If we want to convert this back

$$\begin{align} 2 &\rightarrow 0010 \\ A &\rightarrow 1010 \\ F &\rightarrow 1111 \\ 5 &\rightarrow 0101 \\ 2 &\rightarrow 0010 \\ C &\rightarrow 1100 \\ 3 &\rightarrow 0011 \end{align}$$

$$\begin{align} 0\text{x}\text{2AF52C3} = 0b10101011110101001011000011 \end{align}$$

Conversion With Base-10

The following here is how to convert an unsigned integer (positive, whole number) to hex

If you want to convert a negative integer, see Negative binary, and convert that to hex

To convert from decimal to hex, you need to keep dividing by 16

Divide the number by 16 and store and the result and the remainder

Repeat this process using the result until you get 0

Write the final number from left to right using the remainders from bottom to top

Example: convert $2398$ to hex

$$\begin{align} 2398 / 16 &= 149 \ R \ 14 \ (E_{16}) \\ 149 / 16 &= 9 \ R \ 5 \\ 9 / 16 &= 0 \ R \ 9 \\ \\ 2398_{10} &= 95E_{16} \text{ or } \text{0x95E} \\ \end{align}$$

To convert from hex to decimal, add up each of the terms using the formula $\text{value} \cdot 16^\text{index}$

Don't forget that index starts at zero

For example, convert 0x403B to decimal

$$\begin{align} 11 \cdot 16^0 + 3 \cdot 16^1 + 0 \cdot 16^2 + 4 \cdot 16^3 = 11 + 48 + 0 + 16384 = 16443 \end{align}$$

Practice Problems
  • Convert 0x15A3 to binary and decimal

    1 = 0001 5 = 0101 A = 1010 3 = 0011

    $$\begin{align} \text{0x15A3} = \text{0b0001010110100011} \end{align}$$

    $$\begin{align} \text{0x15A3} &= 3 \cdot 16^0 + 10 \cdot 16^1 + 5 \cdot 16^2 + 1 \cdot 16^3 \\ &= 3 + 160 + 1280 + 4096 = 5539_{10} \end{align}$$

  • Convert 0b100000011110110001

    $0b100000011110110001 \rightarrow 0b00100000011110110001$

    $$\begin{align} 0b00100000011110110001 = 0010 \ 0000 \ 0111 \ 1011 \ 0001 \end{align}$$

    $$\begin{align} 0010 &\rightarrow 2\\ 0000 &\rightarrow 0\\ 0111 &\rightarrow 7\\ 1011 &\rightarrow B\\ 0001 &\rightarrow 1\\ \end{align}$$

    $0b100000011110110001 = \text{0x207B1}$

  • Convert 0xDEAD to binary

    D = 1101 E=1110 A=1010 D = 1101

    $$\begin{align} \text{0xDEAD} = \text{0b1101111010101101} \end{align}$$

  • For a Two's Complement number in hex, how could we easily know the sign?

    For a two's complement number, if the left most digit is 1, it is negative

    The hex characters with a left-most digit with 1 are 8,9,A,B,C,D,E,F

    So if the left-most digit is any of those, it is a negative number

  • What Is The Largest signed int value in hex?

    A signed int is 4 bytes (so 8 hex characters)

    Since we want the largest number, it needs to be positive so the left-most bit can't be one

    The rest of the bits need to be 1 so each term is contributing

    This gives the largest signed int in binary as

    0b01111111111111111111111111111111

    Converting this to hex, we get 0x7FFFFFFF

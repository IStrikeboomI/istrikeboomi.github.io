Just like how we can do Addition in binary, we can also multiply

Similarly, binary multiplication follows a process similar to decimal long multiplication

We can do an even simpler procedure if we are multiplying by power of 2 (1,2,4,8,etc.)

Multiplying By Power of 2 Using Left Shifts

This procedure was covered in Binary Tricks

In decimal, if we want to multiply a number by 10, 100, 1000, etc.

We can simply add zeros to the end of the number (which is moving the whole thing left)

In binary, we can do the same thing but with powers of 2

To multiply by 2, we add one 0 to the end of the number

To multiply by 4, we add two 0s to the end of the number

To multiply by 8, we add three 0s to the end of the number

The process of adding zeros is called a left shift in binary, because it shifts the bits to the left

For example: Multiply $1100101_2$ by $4_{10}$ (or shift binary left by 2)

$$\begin{align} 1100101_2 \times 4_{10} &= 110010100_2 \\ &= 228_{10} \end{align}$$

The operator for a left shift is denoted by <<

So the above operation can be rewritten as $0b1100101 << 2$

You can do a similar thing for right shifts (>>) to divide by powers of 2

Long Multiplication Recap

When you do long multiplication in decimal, you go digit by digit, multiplying the digit by the top number

Then you shift the result to the left by one digit for each digit in the bottom number

Finally, you add all the results together to get the final answer

For example: Multiply 23 by 45

$$\begin{align} 23 \\ \times 4 5 \\ \hline 1 1 5 \\ + 9 2 \ \ \ \\ \hline 1 0 3 5 \\ \end{align}$$

General Binary Multiplication

Binary multiplication works the same way as decimal long multiplication

But there is a benefit because we only have 0s and 1s

When multiplying by 0, the result is always 0, so we can skip that step

When multiplying by 1, the result is always the same as the top number, so we can just copy the top

Another tip is that multiplication is commutative, so pick the number with more 0s on the bottom

For example: Multiply $1101_2$ by $1011_2$ ($13 \times 11 = 143$)

$$\begin{align} 1101 \\ \times 1011 \\ \hline 1101 \\ 1101 \ \ \\ 0000 \ \ \ \ \\ + 1101 \ \ \ \ \ \ \\ \hline 10001111 \\ \end{align}$$

Sometimes when you add the results, you may need to carry bits further than 1 digit place

For example if you need to add $1_2 + 1_2 + 1_2 + 1_2 + 1_2 + 1_2 = 110_2$, you carry a 1 to the 2 next columns

For example multiply $10010101_2$ by $11101011_2$ ($149 \times 235 = 35015 (1000100011000111_2)$)

$$\begin{align} 11101011 \\ \times 10010101 \\ \hline 11101011 \\ 00000000 \ \ \\ 11101011 \ \ \ \ \\ 00000000 \ \ \ \ \ \ \\ 11101011 \ \ \ \ \ \ \ \ \\ 00000000 \ \ \ \ \ \ \ \ \ \ \\ 00000000 \ \ \ \ \ \ \ \ \ \ \ \ \\ + 11101011 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \\ \hline \end{align}$$

Overflow

Similar to addition, you can overflow, which is when the output is too large for the fixed size data type

When this happens, the result is truncated (excess bits is removed)

This means we can ignore the extra digits to add after the data size for easier computation

For example, multiply $1111_2$ and $0011_2$ and store in a 4 bit result

$$\begin{align} 1111 \\ \times 0011 \\ \hline \vert1111 \\ 1\vert111 \ \ \\ 00\vert00 \ \ \ \ \\ + 000\vert0 \ \ \ \ \ \ \\ \hline \text{xxxx}\vert1101 \\ \end{align}$$

Anything beyond the 4 bits is going to be truncated, so we ignore with with an x

Overflow can cause unexpected behavior in computations, such as negative results

It's important to keep overflow in mind when doing binary multiplication to prevent errors

One way to prevent errors is to use a larger output data type.

For example multiplying two 8 bit numbers, the result would be 16 bits

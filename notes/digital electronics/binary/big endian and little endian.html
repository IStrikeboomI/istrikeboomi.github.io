When storing data that is multiply bytes in memory, you have a choice of how to order the bytes

The order in which bytes are stored in memory is called endianness.

In big endian, the most significant byte is stored first (at lowest memory address).

In little endian, the least significant byte is stored first (at lowest memory address).

Each indivual byte however is stored using big endian (MSB comes first within each byte)

Big endian used in our normal base-10/decimal system

On modern computers, most use little endian for storing binary data

Example

Let's say we have the hexadecimal number 0x12345678 (4 bytes)

In big endian, it would be stored in memory as:

Address: 0x00 | 0x01 | 0x02 | 0x03

Value:     0x12 | 0x34 | 0x56 | 0x78

In little endian, it would be stored in memory as:

Address: 0x00 | 0x01 | 0x02 | 0x03

Value:     0x78 | 0x56 | 0x34 | 0x12

You might notice that in little endian, the bytes are reversed compared to big endian.

You might also have noticed we have only done big endian numbers so far

Converting Between Endian Formats

To convert between big endian and little endian, you simply reverse the order of the bytes.

Don't forget in hex, 1 byte is 2 hexadecimal digits (00-FF)

For example, to convert the big endian number 0x12345678 to little endian

You would reverse the order of the bytes to get 0x78563412.

To convert the little endian number 0x78563412 to big endian

You would reverse the order of the bytes to get 0x12345678.

Why Endianness Matters

Endianness matters when interpreting binary data

If you read multi-byte data with the wrong endianness, you'll get incorrect values.

Since we have done everything so far in big endian

You have to convert from little to big endian to do arithmetic properly.

Practice Problems
  • Convert Big Endian of 0b1110001101010101 to Little Endian

    First, split the binary into bytes: 0b11100011 01010101

    Then, reverse the order of the bytes: 0b01010101 11100011

    So, the Little Endian representation is 0b0101010111100011

  • Convert Little Endian of 0xFF00FF88 to Big Endian

    First, split the hexadecimal into bytes: 0xFF 00 FF 88

    Then, reverse the order of the bytes: 0x88 FF 00 FF

    So, the Big Endian representation is 0x88FF00FF

  • Convert Little Endian of 0xFFEA1001 to Decimal

    First, split the hexadecimal into bytes: 0xFF 0xEA 0x10 0x01

    Then, reverse the order of the bytes to get Big Endian: 0x01 0x10 0xEA 0xFF

    Now, combine the bytes back into a single hexadecimal number: 0x0110EAFF

    Finally, convert the hexadecimal number to decimal: 17885951

  • What is the decimal value of the MSB Of 0xB8EEF523 in Big and Little Endian?

    First, split the hexadecimal into bytes: 0xB8 0xEE 0xF5 0x23

    In big endian, the MSB is 0xB8, which is 184 in decimal.

    In little endian, the MSB is 0x23, which is 35 in decimal.
